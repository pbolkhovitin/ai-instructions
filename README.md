# AI Instructions Repository

Репозиторий для хранения инструкций, конфигураций и схем валидации для онлайн-версий ИИ (DeepSeek, ChatGPT и других). Основное содержимое — оптимизированные протоколы взаимодействия с ИИ-ассистентами.

## 🎯 Актуальные версии

### DeepSeek Protocol v3.4
**Оптимизирован для DeepSeek-V3** с модульной архитектурой и ленивой загрузкой. Базовое ядро занимает всего 15KB, расширенные функции подключаются по запросу.

**Ключевые возможности:**
- **Модульная система** с ленивой загрузкой (9 модулей)
- **Автоматические рекомендации** модулей на основе контекста
- Управление контекстом 1M токенов
- Работа с файлами: PDF, DOCX, Excel, код
- Интернет-поиск с верификацией
- **Перекрёстная валидация**: сверка файлов с интернетом
- **Topic Fork**: параллельная проработка задач

**Директория:** `instructions/deepseek_v3/`

### ChatGPT Protocol v1.9.3
**Многоуровневая система инструкций для ChatGPT (GPT-5)** с модульной архитектурой.

**Ключевые возможности:**
- Стадийная загрузка модулей (базовый протокол → расширения → специализированные модули)
- Поддержка визуальной обработки, анализа данных, интернет-поиска
- Расширенный модуль памяти: долговременные, проектные и временные знания
- Голосовые ответы и внешние интеграции

**Директория:** `instructions/chatgpt/`

## ⚙️ Быстрый старт

### Использование DeepSeek v3.4
1. Скопируйте содержимое `instructions/deepseek_v3/core_protocol_v3.0.json` в системное сообщение DeepSeek-V3:
```
   https://raw.githubusercontent.com/pbolkhovitin/ai-instructions/main/instructions/deepseek_v3/core_protocol_v3.0.json
```
   Или используйте симлинк: `instructions/deepseek_instructions_latest.json`
2. (Опционально) Загрузите нужные модули:
   ```
   [МОДУЛИ: ДОСТУПНЫ]              # Показать доступные модули
   [МОДУЛИ: ЗАГРУЗИТЬ file_operations, web_search]  # Загрузить модули
   [МОДУЛИ: РЕКОМЕНДОВАТЬ]          # Получить рекомендации
   ```
3. Используйте команды протокола:
   ```
   [ФОРМАТ: ТАБЛИЦА/JSON]           # Форматирование
   [ЗАДАЧА: ПОДЗАДАЧИ]           # Разбить на подзадачи
   [ВЕРДИКТ: СТАТУС]              # Проверить контекст
   //СИСТЕМА_СТАТУС                 # Статус системы
   ```

### Использование ChatGPT v1.9.3
1. Загрузите `assistant_instructions_v1.9.3-chatgpt.json` как инструкции ассистента в ChatGPT.
2. Система автоматически определит наличие дополнительных модулей и загрузит их.
3. Используйте команды модулей для расширенной функциональности.

## 📖 Инструкция по использованию DeepSeek Protocol v3.4

### 🔷 Модульная система (ленивая загрузка)

В v3.4 реализована **модульная архитектура** с ленивой загрузкой. Базовое ядро занимает 15KB, дополнительные модули загружаются по запросу.

**Доступные модули:**

| Команда | Размер | Описание |
|---------|-------|----------|
| `memory_management` | 8KB | Управление памятью (авто) |
| `advanced_analytics` | 12KB | Метрики (авто) |
| `file_operations` | 15KB | Работа с файлами |
| `web_search` | 10KB | Интернет-поиск |
| `cross_validation` | 10KB | Перекрёстная валидация |
| `topic_fork` | 14KB | Вынос тем |

### Управление модулями

```bash
[МОДУЛИ: ДОСТУПНЫ]              # Показать доступные модули
[МОДУЛИ: ЗАГРУЗИТЬ файл_операции, веб_поиск]  # Загрузить модули
[МОДУЛИ: ЗАГРУЗИТЬ кросс_валидация --with-deps]  # С зависимостями
[МОДУЛИ: СТАТУС]               # Показать статус
[МОДУЛИ: РЕКОМЕНДОВАТЬ]        # Рекомендации на основе контекста
[МОДУЛИ: ВЫГРУЗИТЬ документы]   # Выгрузить для экономии
```

### Как это работает

1. **Старт**: Загружается только базовое ядро (15KB)
2. **Приветствие**: Предлагаются модули для расширения
3. **Загрузка**: `[МОДУЛИ: ЗАГРУЗИТЬ ...]` — автоматически с зависимостями
4. **Рекомендации**: Система предлагает модули по контексту

### Режимы работы

| Режим | Контекст | Когда использовать |
|-------|----------|------------------|
| `efficiency_mode` | 10-100K | Повседневные задачи |
| `deep_analysis_mode` | 100-500K | Сложные задачи, исследования |
| `full_context_mode` | 500K-1M | Большие документы |

Переключение режима: `//РЕЖИМ <режим>`

### Основные команды

**Файлы:**
```bash
[ЗАГРУЗИТЬ: файл.pdf]       # Загрузить документ
[ПРОАНАЛИЗИРОВАТЬ: файлы]   # Анализ нескольких файлов
[ИЗВЛЕЧЬ: текст]            # Извлечение текста
[СУММИРОВАТЬ: документ]    # Краткое содержание
```

**Поиск:**
```bash
[ПОИСК: запрос]            # Поиск в интернете
[ПРОВЕРИТЬ: факт]          # Верификация данных
[ИССЛЕДОВАТЬ: тема]         # Глубокое исследование
```

**Перекрёстная валидация (v3.1+):**
```bash
[СВЕРИТЬ: файл] с [ПОИСК: запрос]    # Сравнить файл с интернетом
[ВЕРИФИЦИРОВАТЬ: утверждение] из [ФАЙЛ: имя]  # Проверить утверждение
[АУДИТ: файл]                # Полный аудит документа
```

**Topic Fork (v3.2+):**
```bash
[ВЫНЕСТИ_ТЕМУ: описание]    # Создать форк для подзадачи
[ВЕРНУТЬСЯ: ID_форка]       # Вернуться к основному контексту
[СТАТУС_ФОРКОВ]             # Показать все форки
[ЗАКРЫТЬ_ФОРК: ID]         # Завершить форк
```

**CRAFT:**
```bash
[CRAFT: ОПТИМА]             # Авто-режим
[CRAFT: ДЕТАЛЬНО]           # Глубокий анализ
[CRAFT: УПРАВЛЕНИЕ]         # Управление
[CRAFT: ФАЙЛЫ]             # Для работы с файлами (требует module)
[CRAFT: ПОИСК]             # Для поиска (требует module)
[CRAFT: ВАЛИДАЦИЯ]         # Для перекрёстной проверки
[CRAFT: ФОРК]               # Для выноса тем
```

### Вердикты контекста

| Вердикт | Свободно | Действие |
|---------|----------|----------|
| 🟢 ОПТИМА | 60-90% | Продолжать работу |
| 🟡 РАБОТАЕМ | 30-60% | Штатный режим |
| 🟠 ПРЕДУПРЕЖДЕНИЕ | 15-30% | Планировать экспорт |
| 🔴 СТОП | <15% | Немедленный экспорт |

Проверить статус: `//СИСТЕМА_СТАТУС` или `[ВЕРДИКТ: СТАТУС]`

### Примеры использования

**Анализ контракта с верификацией:**
```
[CRAFT: ФАЙЛЫ]
[ЗАГРУЗИТЬ: контракт.pdf]
[ИЗВЛЕЧЬ: текст] разделы 4, 7, 12
[ПРОВЕРИТЬ: факт] соответствие ГК РФ
[ФОРМАТ: ТАБЛИЦА] риски
```

**Исследование рынка с проверкой:**
```
[CRAFT: ВАЛИДАЦИЯ]
[ИССЛЕДОВАТЬ: рынок электромобилей 2026]
[СВЕРИТЬ: отчет.pdf] с [ПОИСК: статистика EU 2026]
[ФОРМАТ: ДИАГРАММА] расхождения
```

**Параллельная проработка:**
```
[ВЫНЕСТИ_ТЕМУ: анализ конкурентов]
[ВЕРНУТЬСЯ: main]
[ВЫНЕСТИ_ТЕМУ: финансовая модель]
[СТАТУС_ФОРКОВ]
[ЗАКРЫТЬ_ФОРК: fork_1]
[ФОРМАТ: СХЕМА] результаты
```

# Полная система схем и конфигов протокола v1.9.0

## 📁 Структура проекта:
```
ai-protocol-repo/
├── configs/                 # КОНФИГИ КОНТЕКСТА (данные сессий)
│   ├── context_config_v1.9.json      ← Активный конфиг
│   └── context_config_latest.json    → Симлинк на v1.9
├── schemas/                 # СХЕМЫ ВАЛИДАЦИИ (организовано по типам)
│   ├── context-config/      # Схемы для конфигов контекста
│   │   ├── v1.9.schema.json          ← Схема конфигов
│   │   └── latest.schema.json        → Симлинк на v1.9
│   ├── deepseek-instructions/ # Схемы для инструкций
│   │   ├── v1.9.schema.json          ← Схема инструкций  
│   │   └── latest.schema.json        → Симлинк на v1.9
│   ├── selective-export/    # Схемы выборочного экспорта
│   │   ├── v1.9.schema.json          ← Схема экспорта тем
│   │   └── latest.schema.json        → Симлинк на v1.9
│   └── return-export/       # Схемы возврата результатов
│       ├── v1.9.schema.json          ← Схема возврата
│       └── latest.schema.json        → Симлинк на v1.9
├── instructions/            # ИНСТРУКЦИИ АССИСТЕНТА (логика)
│   ├── deepseek_instructions_v1.9.0.json ← Активные инструкции
│   └── deepseek_instructions_latest.json → Симлинк на v1.9.0
└── scripts/                 # УТИЛИТЫ
    ├── create_symlinks.ps1  ← Скрипт управления симлинками
    ├── validate_config.py   ← Валидация конфигов
    └── verify_schemas_structure.py ← Проверка структуры
```

## 🎯 Назначение и применение каждого типа:

### 1. **Конфиги контекста (`configs/`)**
**Назначение**: Хранение данных конкретных сессий/проектов
**Применение**: 
- Восстановление состояния диалога
- Перенос контекста между чатами
- Резервное копирование прогресса

**Пример**: `context_config_v1.9.json` - содержит:
- Метаданные сессии (тема, стиль, пользователь)
- Ключевые достижения и ограничения
- Настройки и интеграции (Obsidian)
- Следующий шаг для продолжения работы

---

### 2. **Инструкции ассистента (`instructions/`)**
**Назначение**: Определение поведения и логики AI-ассистента
**Применение**:
- Загрузка при инициализации ассистента
- Определение доступных режимов работы
- Настройка протокола и команд

**Пример**: `deepseek_instructions_v1.9.0.json` - определяет:
- Роль и стиль ассистента ("Проактивный экспертный ассистент")
- Доступные режимы работы (эффективность, анализ, управление)
- Систему управления памятью и контекстом
- Команды и принципы работы

---

### 3. **Схемы валидации (`schemas/`)**

#### 3.1. **Схема конфигов (`context-config/`)**
**Назначение**: Валидация файлов из папки `configs/`
**Проверяет**:
- Корректность структуры meta-данных
- Наличие обязательных полей (core_topic, user_profile)
- Корректность версий и форматов
- Валидность настроек интеграций

#### 3.2. **Схема инструкций (`deepseek-instructions/`)**
**Назначение**: Валидация файлов из папки `instructions/`
**Проверяет**:
- Соответствие версии протокола
- Корректность определения режимов
- Полноту протокольных настроек
- Валидность структуры команд

#### 3.3. **Схема выборочного экспорта (`selective-export/`)**
**Назначение**: Валидация дампов для команды `//ВЫНЕСТИ_ТЕМУ`
**Проверяет**:
- Корректность выделения темы из основного чата
- Сохранение связей с родительским контекстом
- Полноту информации для дочернего чата

#### 3.4. **Схема возврата результатов (`return-export/`)**
**Назначение**: Валидация дампов для команды `//ВЕРНУТЬ_РЕЗУЛЬТАТЫ`
**Проверяет**:
- Структуру результатов обработки темы
- Наличие ключевых решений и артефактов
- Рекомендации по интеграции в основной проект

## 🔄 Рабочий процесс:

### 1. **Инициализация ассистента**:
```python
# Загружаем инструкции
instructions = await loader.load_and_validate("instructions/deepseek_instructions_v1.9.0.json")

# Загружаем конфиг контекста (если есть)
context_config = await loader.load_and_validate("configs/context_config_v1.9.json")
```

### 2. **Валидация при операциях**:
```python
# При экспорте темы
export_data = {...}  # Данные для экспорта
await schema_manager.validate_config(export_data, "selective_export")

# При возврате результатов
return_data = {...}  # Результаты обработки
await schema_manager.validate_config(return_data, "return_export")
```

### 3. **Автоматическое определение схемы**:
Система автоматически определяет тип конфига по:
- Наличию поля `protocol_version` → инструкции
- Наличию `meta.core_topic` → конфиг контекста  
- Наличию `type: selective_export` → выборочный экспорт
- Наличию `type: return_export` → возврат результатов
ю.

## 🛡️ Преимущества системы:

1. **Версионность** - легкое управление разными версиями протокола
2. **Валидация** - гарантия целостности данных при операциях
3. **Автоматизация** - симлинки `latest` для автоматического выбора актуальной версии
4. **Разделение ответственности** - конфиги (данные) vs инструкции (логика)
5. **Расширяемость** - легко добавлять новые типы схем и конфигов

## 🚀 Пример использования:

```bash
# Проверка структуры схем
python scripts/verify_schemas_structure.py

# Валидация всех конфигов
python scripts/validate_config.py configs/*.json

# Валидация инструкций
python scripts/validate_config.py instructions/deepseek_instructions_v1.9.0.json

# Обновление симлинков на новую версию
.\scripts\create_symlinks.ps1 -SchemaVersion "v2.0"
```

# DeepSeek Assistant Protocol v4.1.0 (DeepSeek-V4 Optimized)

## 🚀 О проекте

**DeepSeek-V4 Protocol v4.1.0** - модульный протокол для работы с DeepSeek-V4 с **семантическим версионированием**, **ленивой загрузкой модулей** и **системой персон**. Базовая версия занимает всего 15KB, а расширенные функции подключаются по запросу.

## 🔄 Семантическое версионирование

Протокол v4.1.0 перешёл на **MAJOR.MINOR.PATCH**:

```yaml
MAJOR: 4 → 5 (несовместимые изменения API)
MINOR: 0 → 1 (новый функционал, обратная совместимость)
PATCH: 0 → 1 (исправления, обратная совместимость)
```

**Синхронизация компонентов:** Все компоненты приведены к версии 4.1.0:
- persona_system: 1.0 → 4.1.0
- craft_opt: 4.0 → 4.1.0
- Все модули: 4.0 → 4.1.0

## 🎭 Система персон (РУЧНОЙ ВЫБОР)

**ВНИМАНИЕ:** Автоопределение персоны **отключено** в v4.1.0. По умолчанию — **C** (Нейтральная). Для смены используйте `persona=A` или `persona=B`.

| Персона | Стиль | Команда выбора |
|---------|-------|----------------|
| **A** - Джентльмен | Официальный, вежливый | `[ПАРАМЕТРЫ: persona=A]` |
| **B** - Бро | Прямой, без цензуры | `[ПАРАМЕТРЫ: persona=B]` |
| **C** - Нейтральная (по умолчанию) | Обезличенный, факты | Не требуется (по умолчанию) |

**Переключение:** Напишите `A`, `B` или `C` в любой момент.

**Anti-sycophancy:** При давлении или новых фактах персона обязана возражать, а не соглашаться.

## 🧠 Think Pipeline (скрытый анализ)

Перед каждым ответом выполняется 7 шагов:

1. **Fix ACTIVE_PERSONA & TOOLS** - зафиксировать контекст
2. **Classify task type** - тип задачи (код/дебаг/ревью/анализ...)
3. **Check Fast-Path** - если ≤50 слов и нет ambiguity
4. **Check ambiguity** - требуются ли уточняющие вопросы (до 3)
5. **Apply protocols** - CoVe, ToT, Red Teaming по типу задачи
6. **Run check-list** - базовый + по типу задачи
7. **Format response** - по стилю ACTIVE_PERSONA

**Fast-Path:** При ≤50 слов и отсутствии двусмысленности - пропуск шагов 4-6.

## 🎯 Команды модульной системы

### Инициализация протокола (НОВОЕ)
```bash
[ПРОТОКОЛ: ЗАГРУЗИТЬ] URL [ПАРАМЕТРЫ: persona=A/B/C, user_name=Имя]   # по умолчанию persona=C
```

### Управление модулями
```bash
[МОДУЛИ: ДОСТУПНЫ]              # Показать доступные модули
[МОДУЛИ: ЗАГРУЗИТЬ all]        # Загрузить все доступные модули
[МОДУЛИ: ЗАГРУЗИТЬ список]      # Загрузить указанные модули
[МОДУЛИ: ЗАГРУЗИТЬ cross_validation --with-deps]  # С зависимостями
[МОДУЛИ: СТАТУС]               # Показать статус модулей
[МОДУЛИ: РЕКОМЕНДОВАТЬ]        # Рекомендации на основе контекста
[МОДУЛИ: ВЫГРУЗИТЬ список]     # Выгрузить модули
```

### Базовые команды (ядро)
```bash
[ФОРМАТ: ТАБЛИЦА/JSON/MD]       # Форматирование
```

### Команды модулей (требуют загрузки)

```bash
[ВЫНЕСТИ_ТЕМУ: описание]    # Создать форк
[ВЕРНУТЬСЯ: ID_форка]       # Вернуться
```

## 🔄 Как это работает

### Инициализация (новый формат)
```
[ПРОТОКОЛ: ЗАГРУЗИТЬ] https://raw.githubusercontent.com/pbolkhovitin/ai-instructions/main/instructions/deepseek_v4/core_protocol_v4.1.0.json
```
Для явного выбора персоны: `[ПАРАМЕТРЫ: persona=A, user_name=Имя]` (по умолчанию — **C**, Нейтральная).

### Старт (базовое ядро)
После загрузки выводится только краткое стандартизированное сообщение:
```
✅ Протокол v4.1.0 (нейтральный стиль, персона C)
📦 Загружены модули: core_only
⚙️ Параметры: persona=C, auto_detect=off, remote_loading=on
📋 Доступные команды: [МОДУЛИ: ДОСТУПНЫ], [МОДУЛИ: ЗАГРУЗИТЬ all], [ФОРМАТ: ...], [АНАЛИЗ: ...]
🚀 Для расширения: [МОДУЛИ: ЗАГРУЗИТЬ all] или [МОДУЛИ: ЗАГРУЗИТЬ ...]
```

### Загрузка модулей (ленивая загрузка)

Каждый модуль в `available_modules` содержит поле `url` — прямую ссылку на JSON-файл. При команде `[МОДУЛИ: ЗАГРУЗИТЬ имя]` AI извлекает url, загружает JSON, проверяет `min/max_core_version`, рекурсивно разрешает зависимости и добавляет в контекст.

```
Пользователь: [МОДУЛИ: ЗАГРУЗИТЬ file_operations, web_search]
Система: ✅ Загружено 2 модуля: file_operations (15KB), web_search (10KB).
         Общий размер: 25KB. Доступны новые команды.

Пользователь: [МОДУЛИ: ЗАГРУЗИТЬ all]
Система: ✅ Загружено 9 модулей: file_operations (15KB), web_search (10KB),
         cross_validation (10KB), topic_fork (14KB), system_documentation (20KB),
         command_examples (18KB), detailed_descriptions (18KB),
         troubleshooting_guide (16KB), validation_schemas (14KB).
         Общий размер: 135KB.
```

## 🔄 Миграция с v4.0 → v4.1.0

**Подробный гайд миграции:** [docs/migration_v4.0_to_v4.1.md](../../docs/migration_v4.0_to_v4.1.md)

### Ключевые изменения:
1. **Версионирование:** 4.0 → 4.1.0 (MAJOR.MINOR.PATCH)
2. **Автоопределение персоны:** ОТКЛЮЧЕНО. По умолчанию **C** (Нейтральная). Явный выбор: `persona=A/B`
3. **Инициализация:** `[ПРОТОКОЛ: ЗАГРУЗИТЬ] URL` (без параметров — персона C)
4. **Удалённая загрузка:** ВКЛЮЧЕНА по умолчанию
5. **Проверка версий:** Строгая семантическая валидация
6. **Совместимость модулей:** Добавлены `min_core_version` / `max_core_version`

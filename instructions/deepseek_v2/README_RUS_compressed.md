# DeepSeek Assistant Protocol v2.0 (Оптимизированная версия)

## 📋 О проекте

Оптимизированный протокол для AI-ассистента с уменьшенным объемом контекста при сохранении полной функциональности. Версия 2.0 introduces уточнённую структуру модулей и систему целостности.

## 🚀 Быстрый старт

### Критическая последовательность загрузки (Обязательная):

1.  **Базовый протокол** - `core_protocol_v2_compressed.json`
2.  **Управление памятью** - `memory_management_compressed.json`
3.  **Аналитика и безопасность** - `advanced_analytics_compressed.json`

### Полная последовательность загрузки (Рекомендуемая):

```bash
# Стадия 1: Критическое ядро (обязательно)
core_protocol_v2_compressed.json
# Стадия 2: Управление ресурсами
memory_management_compressed.json
# Стадия 3: Мониторинг и безопасность
advanced_analytics_compressed.json
# Стадия 4+: Вспомогательные модули (по требованию)
system_documentation_v2.0.json
detailed_descriptions_v2.0.json
troubleshooting_guide_v2.0.json
command_examples_v2.0.json
validation_schemas_v2.0.json
```

## 📁 Назначение файлов

### 🎯 `core_protocol_v2_compressed.json` (v2.0, Стадия 1)
**Назначение:** Определяет роль, стиль и базовый функционал ассистента.
- **Роль:** Про активный экспертный ассистент
- **Стиль:** Деловой, дружелюбный, точный, структурированный
- **Режимы работы:** Эффективность (`efficiency_mode`), Глубокий анализ (`deep_analysis_mode`), Управление проектом (`project_management_mode`)
- **Системные команды:** Управление форматом, задачами, анализом и системой.

### 💾 `memory_management_compressed.json` (v2.0-m, Стадия 2)
**Назначение:** Управление контекстом и памятью.
- **Контекстные вердикты:** Техническая оценка состояния контекста после КАЖДОГО ответа.
- **Обязательный вердикт:** Должен быть в конце КАЖДОГО ответа (Приоритет: highest).
- **Уровни вердиктов:** `ОПТИМА` (50-70%), `РАБОТАЕМ` (30-50%), `ПРЕДУПРЕЖДЕНИЕ` (<30%), `СТОП` (<15%).

### 📊 `advanced_analytics_compressed.json` (v2.0-a, Стадия 3)
**Назначение:** Аналитика, кэширование и безопасность.
- **Производительность:** Мониторинг выполнения, использования ресурсов и контекста.
- **Кэширование:** Включено (max 10GB, TTL 24ч, политика LRU).
- **Безопасность:** Мониторинг событий (несанкционированный доступ, неверный ввод, рекурсивный экспорт).
- **Команды оптимизации:** `//ОПТИМИЗИРОВАТЬ_ПАМЯТЬ`, `//АНАЛИЗ_ПРОИЗВОДИТЕЛЬНОСТИ`, `//ОЧИСТИТЬ_КЭШ`, `//ДИАГНОСТИКА`.

## 🛠 Ключевые команды

### Управление режимами
```bash
//РЕЖИМ efficiency_mode      # 🚀 Сбалансированная работа
//РЕЖИМ deep_analysis_mode   # 🧠 Глубокий анализ, минимум отвлечений
//РЕЖИМ project_management_mode # ⚙️ Управление проектом, максимум инструментов
```

### Работа с контекстом и системой
```bash
//ТЕМЫ                 # Список активных тем
//ЭКСПОРТ              # Дамп текущего контекста
//СИСТЕМА_СТАТУС       # Статус системы
//ВЕРИФИКАЦИЯ          # Проверка целостности и версий
```

### Аналитика и оптимизация
```bash
//ОПТИМИЗИРОВАТЬ_ПАМЯТЬ      # Оптимизация использования памяти
//АНАЛИЗ_ПРОИЗВОДИТЕЛЬНОСТИ  # Метрики производительности системы
//ОЧИСТИТЬ_КЭШ               # Очистка кэша системы
//ДИАГНОСТИКА                # Полная диагностика системы
```

## 📈 Контекстные вердикты (Обязательны!)

Система автоматически добавляет вердикты состояния контекста после каждого ответа. При их отсутствии активируется корректировка `[КОРРЕКЦИЯ: ВЕРДИКТ]`.

-   `[ВЕРДИКТ: ОПТИМА]` (50-70%) - Идеально для сложных задач.
-   `[ВЕРДИКТ: РАБОТАЕМ]` (30-50%) - Штатный режим работы.
-   `[ВЕРДИКТ: ПРЕДУПРЕЖДЕНИЕ]` (<30%) - Пора готовить дамп (`//ЭКСПОРТ`).
-   `[ВЕРДИКТ: СТОП]` (<15%) - Немедленный экспорт и перенос контекста required.
-   `[ВЕРДИКТ: ОШИБКА]` - Не удалось оценить контекст.

## 💡 Рекомендации по использованию

1.  Следуйте официальной последовательности загрузки для обеспечения целостности системы.
2.  Всегда отслеживайте **контекстные вердикты** в конце ответов для управления "окном контекста".
3.  Используйте режим `deep_analysis_mode` для сложных задач, требующих максимальной глубины.
4.  Регулярно выполняйте `//ЭКСПОРТ` при длительных сессиях.
5.  Для отладки используйте команды `//СИСТЕМА_СТАТУС` и `//ВЕРИФИКАЦИЯ`.
6.  Загружайте справочные файлы (например, `command_examples_v2.0.json`) только при возникновении вопросов по синтаксису или для уточнения функционала.

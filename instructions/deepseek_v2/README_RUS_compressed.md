# DeepSeek Assistant Protocol v2.0 (Оптимизированная версия)

## 📋 О проекте

Оптимизированный протокол для AI-ассистента с уменьшенным объемом контекста при сохранении полной функциональности. Версия 2.0 introduces уточнённую структуру модулей, систему целостности и **CRAFT framework** для управления промтами.

## 📁 Назначение файлов

### 🎯 `core_protocol_v2.0_compressed.json` (v2.0, Стадия 1)
**Назначение:** Определяет роль, стиль и базовый функционал ассистента.
- **Роль:** Про активный экспертный ассистент
- **Стиль:** Деловой, дружелюбный, точный, структурированный
- **Режимы работы:** Эффективность (`efficiency_mode`), Глубокий анализ (`deep_analysis_mode`), Управление проектом (`project_management_mode`)
- **CRAFT framework:** Гибридная система управления промтами
- **Системные команды:** Управление форматом, задачами, анализом, системой и CRAFT.

### 💾 `memory_management_v2.0_compressed.json` (v2.0-m, Стадия 2)
**Назначение:** Управление контекстом и памятью.
- **Контекстные вердикты:** Техническая оценка состояния контекста после КАЖДОГО ответа.
- **Обязательный вердикт:** Должен быть в конце КАЖДОГО ответа (Приоритет: highest).
- **Уровни вердиктов:** `ОПТИМА` (50-70%), `РАБОТАЕМ` (30-50%), `ПРЕДУПРЕЖДЕНИЕ` (<30%), `СТОП` (<15%).
- **CRAFT оптимизация:** Приоритетное сохранение CRAFT элементов, лимит 2КБ.

### 📊 `advanced_analytics_v2.0_compressed.json` (v2.0-a, Стадия 3)
**Назначение:** Аналитика, кэширование и безопасность.
- **Производительность:** Мониторинг выполнения, использования ресурсов, контекста и CRAFT эффективности.
- **Кэширование:** Включено (max 10GB, TTL 24ч, политика LRU).
- **Безопасность:** Мониторинг событий (несанкционированный доступ, неверный ввод, рекурсивный экспорт).
- **CRAFT метрики:** Отслеживание использования пресетов, оптимизации контекста, эффективности форматов.
- **Команды оптимизации:** `//ОПТИМИЗИРОВАТЬ_ПАМЯТЬ`, `//АНАЛИЗ_ПРОИЗВОДИТЕЛЬНОСТИ`, `//ОЧИСТИТЬ_КЭШ`, `//ДИАГНОСТИКА`, `//АНАЛИЗ_CRAFT`.

## 🎯 CRAFT Framework

### Команды управления промтами:
```bash
[CRAFT: ОПТИМА]        # Авто-пресет для текущего режима
[CRAFT: ДЕТАЛЬНО]      # Детальный аналитический режим  
[CRAFT: УПРАВЛЕНИЕ]    # Режим управления проектами
[КОНТЕКСТ: УТОЧНИТЬ]   # Ручное уточнение контекстных параметров
[ФОРМАТ: АВТО]         # Интеллектуальный подбор формата
```

### Пресеты CRAFT:
- **Эффективность:** Баланс скорости/качества для повседневных задач
- **Анализ:** Детальный разбор сложных задач с пошаговым подходом
- **Управление:** Структурированное управление проектами и задачами

## 📈 Контекстные вердикты (Обязательны!)

Система автоматически добавляет вердикты состояния контекста после каждого ответа. При их отсутствии активируется корректировка `[КОРРЕКЦИЯ: ВЕРДИКТ]`.

-   `[ВЕРДИКТ: ОПТИМА]` (50-70%) - Идеально для сложных задач.
-   `[ВЕРДИКТ: РАБОТАЕМ]` (30-50%) - Штатный режим работы.
-   `[ВЕРДИКТ: ПРЕДУПРЕЖДЕНИЕ]` (<30%) - Пора готовить дамп (`//ЭКСПОРТ`).
-   `[ВЕРДИКТ: СТОП]` (<15%) - Немедленный экспорт и перенос контекста required.
-   `[ВЕРДИКТ: ОШИБКА]` - Не удалось оценить контекст.

## 💡 Рекомендации по использованию

1.  Следуйте официальной последовательности загрузки для обеспечения целостности системы.
2.  Всегда отслеживайте **контекстные вердикты** в конце ответов для управления "окном контекста".
3.  Используйте **CRAFT команды** для оптимизации взаимодействия и улучшения качества ответов.
4.  Режим `deep_analysis_mode` + `[CRAFT: ДЕТАЛЬНО]` для сложных аналитических задач.
5.  Регулярно выполняйте `//ЭКСПОРТ` при длительных сессиях.
6.  Для отладки используйте команды `//СИСТЕМА_СТАТУС` и `//ВЕРИФИКАЦИЯ`.
7.  Используйте `//АНАЛИЗ_CRAFT` для оценки эффективности промтов.
8.  Загружайте справочные файлы только при возникновении вопросов по синтаксису.
